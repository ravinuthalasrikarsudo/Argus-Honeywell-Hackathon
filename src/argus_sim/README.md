# argus_sim — Gazebo worlds & drone model (Pillar 1)

Simulation assets for the GNSS-denied indoor environments, resolved at runtime
through the package's `GZ_SIM_RESOURCE_PATH` hook.

| Asset | Description |
|-------|-------------|
| `worlds/warehouse_corridor.sdf` | Frozen Day-1 contract world: 30×5×3 m warehouse corridor, zones A/B/C, slalom obstacles, racking, Zone-B flicker light. Generated by `worlds/generate_world.py`. |
| `worlds/tunnel_circuit.sdf` | Scenario E world: 202.8 m closed-circuit stadium tunnel (2×70 m straights + r=10 m end-caps) for the 200 m drift gate. Generated by `worlds/generate_tunnel_circuit.py`. |
| `worlds/detail.png` | PBR checker+speckle albedo texture — the deliberate feature-richness layer for the VIO front-end. Referenced relative to the world file. |
| `models/argus_drone/model.sdf` | Kinematic quadrotor: stereo pair (0.12 m baseline, 15 Hz), 250 Hz IMU, 3D LiDAR, downward rangefinder, VelocityControl + PosePublisher. |

The worlds are **generated** — edit the `generate_*.py` script, re-run it, then
rebuild this package (`colcon build --packages-select argus_sim`); never edit
the `.sdf` by hand.

Launch via `argus_bringup`:

```bash
ros2 launch argus_bringup argus_sim.launch.py headless:=true                       # corridor
ros2 launch argus_bringup argus_sim.launch.py headless:=true world:=tunnel_circuit # 200 m tunnel
```

The full sensor/topic specification is the frozen contract in `docs/CONTRACT.md`.
